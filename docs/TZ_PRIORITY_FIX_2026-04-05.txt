БЛОК 1 — УБРАТЬ СВЯЗЬ ПО ИМЕНИ ПОЗИЦИИ
--------------------------------------------------------------------------------
Проблема:
- используется fallback: SKU / имя / совпадение текста
- возможны ошибки при одинаковых или похожих названиях

Требование:
- использовать ТОЛЬКО:
  - inventory_item_id
  - order_warehouse_line_id
- запретить использование name как бизнес-логики

Разрешено:
- поиск по имени только как UI-подсказка

БЛОК 2 — ЦЕНЫ НЕ ДОЛЖНЫ ТЕРЯТЬСЯ
--------------------------------------------------------------------------------
Проблема:
- purchase_price = 0
- sale_price = 0 после попадания в склад

Требование:
- гарантировать передачу:
  order_lines_json → order_warehouse_lines → inventory

Проверить:
- mapping полей
- создание позиций
- приход

Запрещено:
- создавать складскую позицию с нулевой ценой, если она была указана

================================================================================

БЛОК 3 — ФИНАНСЫ ДОЛЖНЫ БЫТЬ ЛОГИЧЕСКИ КОРРЕКТНЫМИ
--------------------------------------------------------------------------------
Проблема:
- расход учитывается, доход — частично
- прибыль отображается некорректно

Требование:
- доход = продажа позиций + работа
- расход = закупка
- прибыль = доход - расход

Если есть order_lines_json:
- считать по строкам
