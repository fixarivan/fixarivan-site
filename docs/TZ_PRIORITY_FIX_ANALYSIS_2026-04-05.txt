    сопоставление строки заказа с order_lines_json по имени; fallback «одна
    позиция в inventory_items по имени».
  • fixarivan_find_order_warehouse_line_for_arrival — имя + fuzzy match.
  • fixarivan_resolve_inventory_item_id_for_deduction — имя / fuzzy.
  • api/order_purchase_queue_action.php — dismiss по (order_id, name_norm).
  • api/order_purchase_queue.php — фильтр dismiss по name_norm.
  • inventory_movement.php — при отсутствии owl_id поиск по имени товара.

Уже в сторону ТЗ:
  • order_lines_json при сохранении из Track обогащается inventory_item_id по SKU
    (fixarivan_enrich_order_lines_inventory_ids) — ключ SKU, не имя.
  • Движение с явным order_warehouse_line_id.

План работ (минимальный для Блока 1):
  1) Хранить в order_warehouse_lines стабильный ключ связи (уже есть id строки);
     привязку к складу — только через inventory_item_id из JSON/синка, без
     «угадывания» по имени в PHP.
  2) Убрать или ограничить: resolve по имени; fuzzy в supply — заменить на
     явный id или отказ в операции.
  3) Dismiss списка покупок: заменить на owl_id-only (или хранить owl_id в
     dismiss), чтобы не зависеть от имени.

Риск: миграция данных и существующие заказы без SKU — потребуется
     одноразовое заполнение inventory_item_id в Track или ручная правка.

Основные файлы: api/lib/order_supply.php, api/lib/order_warehouse_sync.php,
  api/order_purchase_queue.php, api/order_purchase_queue_action.php,
  api/inventory_movement.php, api/order_track_order_lines.php.
