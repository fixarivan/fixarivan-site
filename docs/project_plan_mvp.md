# План MVP (этапы 1-7)

## 1) Анализ
- Сущности: документы (`order`, `receipt`, `report`) и события календаря.
- Хранилище: только SQLite для событий (`calendar_events`), без дубля в MySQL.
- Связь: событие может ссылаться на документ через `link_type` + `link_id`.

## 2) План API и экранов
- API событий: `api/calendar_events.php`
  - `GET summary=1` — сводка для дашборда
  - `GET month=YYYY-MM` — события месяца
  - `GET event_id=...` — одно событие для диплинка
  - `POST action=save|delete` — запись/удаление
- Экраны:
  - `dashboard_app.html` — центр управления + сводка событий
  - `calendar.html` — месяц/день + форма события

## 3) PWA
- `manifest.json` + `assets/icons/icon.svg`.
- Регистрация service worker (`sw.js`) на дашборде и в календаре.

## 4) Mobile UX
- Упрощённые элементы управления в календаре:
  - крупные кнопки, переключатель вида, список дня.
- Без drag-and-drop, с фокусом на быстрые действия.

## 5) События (SQLite)
- Таблица `calendar_events` + индексы в `api/sqlite.php`.
- Логика выборок/сводки в `api/lib/calendar_events.php`.

## 6) Календарь
- `calendar.html`: сетка месяца, список дня, фильтр по статусу.
- Форма: создание/редактирование, статус, заметки, связь с документом.

## 7) Dashboard
- `dashboard_app.html`: блок "События" (просроченные/сегодня/ближайшие).
- Быстрые действия и диплинки на конкретное событие в календаре.
