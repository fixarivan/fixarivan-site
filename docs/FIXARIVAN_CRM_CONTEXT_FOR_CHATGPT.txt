1. ЧТО ЭТО ЗА ПРОДУКТ
================================================================================

FixariVan — веб-CRM для мобильного сервиса по ремонту техники (Финляндия).
Один мастер / одна мастерская, shared-хостинг, SQLite как единая БД.

Объединяет в одном месте:
  • Заказы на ремонт и продажу
  • Акт приёма, квитанции, счета (invoice), диагностические отчёты (mobile/PC)
  • Клиентскую базу
  • Склад запчастей с движениями и привязкой к заказам
  • Календарь событий
  • Финансовую аналитику (кассовый метод, не полноценная бухгалтерия)
  • Публичный клиентский портал по токену в URL
  • PDF-документы (Dompdf)
  • Заметки мастера

Production URL: https://fixarivan.space/
GitHub: fixarivan/fixarivan-site (ветка main)

2. ТЕХНОЛОГИИ
================================================================================

Сервер:
  • PHP 8.x
  • PDO SQLite (файл storage/fixarivan.sqlite)
  • PHP-сессии для админки (cookie_path=/, SameSite=Lax, HttpOnly)
  • Dompdf для PDF (api/dompdf/)

Клиент:
  • HTML + CSS + vanilla JavaScript (fetch API)
  • Без React/Vue — монолитные .html страницы + inline/отдельные .js
  • Chart.js 4.4 (CDN) — графики на дашборде и складе
  • Service Worker (sw.js) — кэш статики, API НЕ кэшируется

Конфигурация:
  • config.example.php → config.local.php (НЕ коммитить!)
  • ADMIN_USERNAME / ADMIN_PASSWORD в config.local.php
  • После смены пароля в настройках — storage/admin_auth.json (bcrypt)
  • storage/company_profile.json — реквизиты компании, путь к логотипу

Legacy (не использовать для новых фич):
  • MySQL в config.php — устарело
  • api/get_statistics_optimized.php — deprecated MySQL
  • admin/dashboard.html — старая панель с hardcoded логином

4. БАЗА ДАННЫХ SQLite
================================================================================

Файл: storage/fixarivan.sqlite
Инициализация/миграции: api/sqlite.php → ensureSqliteSchema()
Режим: WAL, foreign keys, busy_timeout

Основные таблицы:

clients          — клиенты (имя, телефон, email, …)
orders           — заказы / акты приёма
  • status (legacy: pending, signed, sent_to_client, …)
  • order_status, public_status (актуальная модель Track)
  • parts_status — статус запчастей по заказу
  • order_lines_json — позиции заказа (запчасти, работы)
  • client_token — для публичных ссылок
  • date_of_acceptance, date_created, date_updated, public_completed_at

receipts         — квитанции об оплате
  • payment_status, payment_date, total_amount, order_id

invoices         — счета B2B
  • status (draft, sent, paid, cancelled, …)

mobile_reports   — диагностические отчёты (mobile/PC)

inventory_items  — каталог склада (sku, name, category, default_cost, sale_price, min_stock)
inventory_balances — остаток quantity на item_id (триггер из movements)
inventory_movements — ledger: sale, out, in, adjust… quantity_delta, ref_kind, ref_id

calendar_events  — события календаря
order_warehouse_lines — связь строк заказа со складом
purchase_list_dismissals — очередь закупок
notes            — быстрые заметки мастера
portal_snake_global — таблица для мини-игры в клиентском портале

Файлы вне git (на сервере, сохраняются при deploy):
  • storage/fixarivan.sqlite
  • storage/company_profile.json
  • storage/admin_auth.json
  • assets/company_logo.* — загруженный логотип

6.1. Рабочий стол (dashboard_app.html)

7. СТАТИСТИКА И РАСЧЁТЫ (ВАЖНО ДЛЯ AI)
================================================================================

Источник: api/lib/dashboard_stats.php (вызывается из get_fast_stats.php)

Заказы — bucket по public_status/order_status + legacy fallback:
  waiting      → waiting_parts + legacy pending/draft
  in_progress  → in_progress, in_transit + legacy sent_to_client/viewed
  completed    → done, delivered + legacy signed
  cancelled    → cancelled

Фильтр периода: date_of_acceptance ?? work_date ?? date_created

total на дашборде = заказы за период (НЕ receipts+reports; total_documents — legacy поле)

Склад (sqliteInventoryAggregateStats):
  in_stock     → quantity > 0
  out_of_stock → total − in_stock
  low_stock    → qty > 0 AND qty <= min_stock
  profit_potential → sum(qty * sale_price) − sum(qty * default_cost)

Финансы на дашборде — через fixarivan_finance_overview() (finance_lib.php)

НЕ смешивать:
  • KPI заказов (текущая модель Track)
  • deprecated get_statistics_optimized.php (MySQL statuses)

  • CSS grid-слот: .fv-brand-logo-slot — custom заменяет default
  • Класс has-custom-brand-logo + :has() селекторы в fixarivan-theme.css

Рекомендации загрузки (актуально):
  • PNG прозрачный, горизонтальный, ~1200×320 px (пропорция 3.5:1…4:1)
  • Лимит файла: 4 МБ
  • Пиксели: 240×64 … 2400×800
  • На экране: до 480×128 px (дашборд), 280×72 px (склад)
  • object-fit: contain — пропорции сохраняются

Валидация: fixarivan_company_logo_validate_upload() в company_profile.php

  ✓ Новые таблицы/колонки — через ensureSqliteSchema в sqlite.php
  ✓ Сохранять обратную совместимость API-полей где возможно

  ✗ Переписывать на React/фреймворк
